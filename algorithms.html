Enhancement Two: Algorithms and Data Structures

Overview

This enhancement focuses on improving the efficiency and structure of data handling in the Travlr full-stack application. The goal was to make the project more effective by applying stronger algorithmic thinking, improving data access patterns, and using more structured logic for handling trip data.

Artifact Description

The artifact used for this enhancement is the Travlr full-stack application developed in CS 465. The project manages trip information through a full-stack architecture, and this enhancement emphasized improved data handling, structured problem solving, and more efficient processing of trip-related operations.

Enhancements Implemented

  • Improved the organization of trip-processing logic so that data flows more predictably through the application
  • Used structured validation and normalization steps to reduce unnecessary reprocessing of inconsistent data
  • Improved search and lookup efficiency by supporting indexed access through trip codes
  • Applied more deliberate logic to create and update operations so that invalid or duplicate entries are detected earlier
  • Strengthened the overall handling of trip collections by making data operations more consistent and scalable

Algorithmic and Data Structure Considerations

  • Validation Before Storage: Data is checked and normalized before database operations, reducing avoidable processing later
  • Indexed Lookup: Trip code indexing improves retrieval efficiency compared to scanning larger collections manually
  • Duplicate Detection: Logic was added to detect conflicting trip codes before insert or update operations complete
  • Structured Data Flow: The enhancement improves how data moves through the system, making the logic easier to reason about

Impact of Enhancements

  • Efficiency: Data lookups and validation workflows are more efficient and reliable
  • Problem Solving: The enhancement demonstrates the use of algorithmic reasoning to improve backend logic
  • Consistency: Data is processed more uniformly, reducing edge-case failures
  • Extensibility: The application is better prepared for future growth in the number of trips and related operations

Course Outcomes Demonstrated

  • Outcome 1: Supported decision making by organizing data operations in a clearer and more predictable way
  • Outcome 2: Communicated technical improvements through documentation and structured explanation of logic changes
  • Outcome 3: Applied algorithmic principles and design trade-offs to improve how the application handles data
  • Outcome 4: Used appropriate tools and computing practices to improve the application's efficiency and usefulness
  • Outcome 5: Reduced risk by ensuring invalid or conflicting data is handled earlier in the workflow

Reflection

This enhancement helped me better understand how algorithmic thinking applies outside of traditional standalone programs. I learned that efficiency, validation order, and lookup strategy all affect how well a full-stack application performs. This process showed me that even small structural improvements in logic and data handling can make a meaningful difference in reliability, scalability, and maintainability.
