Enhancement Three: Databases

Overview

This enhancement focused on improving the database layer of the Travlr full-stack application. The goal was to strengthen data validation, improve data integrity, and optimize how data is stored and accessed.

Artifact Description

The artifact used for this enhancement is the Travlr full-stack application developed in CS 465. This application uses MongoDB, Express, Node.js, and Angular to manage trip data through RESTful API endpoints.

Enhancements Implemented

  • Enhanced tripServices.js with data normalization, including trimming strings, converting price values to numbers, and uppercasing trip codes
  • Added validation logic for required fields, valid dates, and non-negative pricing
  • Implemented duplicate trip code prevention during create and update operations
  • Updated the travlr.js schema:
    • Converted perPerson from String to Number
    • Added validation rules such as required fields, minimum length, and minimum numeric values
    • Maintained an index on trip code for performance optimization
  • Refactored the trips.js controller:
    • Used validated payloads instead of raw request data
    • Improved error handling with clearer 400, 404, and 500 responses

Impact of Enhancements

  • Data Integrity: Prevents invalid or incomplete data from being stored in the database
  • Performance: Indexed queries improve lookup speed and support scalability
  • Security: Validation reduces the risk of malformed or harmful input reaching the database layer
  • Maintainability: Service-layer validation and cleaner controller logic improve code organization

Course Outcomes Demonstrated

  • Outcome 1: Supported structured data workflows that enable more reliable decision making through consistent validation and data handling
  • Outcome 2: Communicated database improvements through organized backend structure and clear technical documentation
  • Outcome 3: Designed efficient data-handling solutions using validation logic, normalization, and indexed queries
  • Outcome 4: Applied industry tools such as MongoDB and Mongoose to implement a more scalable and reliable database solution
  • Outcome 5: Demonstrated a security mindset by enforcing validation, preventing duplicate records, and improving error handling

Reflection

This enhancement improved my understanding of how database design impacts performance, reliability, and security. I learned the importance of validating and normalizing data before it reaches the database and ensuring that backend changes do not break existing functionality. This process reinforced the value of designing database-backed applications that are both scalable and secure.
