Enhancement Three: Databases

Overview

This enhancement focuses on improving the database layer of the Travlr application by strengthening validation, data integrity, and performance.

Artifact Description

The Travlr application uses MongoDB and Mongoose to manage trip data through RESTful APIs.

Enhancements Implemented

  • Data normalization in tripServices.js
  • Validation logic for required fields and pricing
  • Duplicate prevention
  • Schema improvements and indexing
  • Improved controller error handling

Impact of Enhancements

  • Data Integrity: Prevents invalid data
  • Performance: Faster queries via indexing
  • Security: Input validation reduces risk
  • Maintainability: Cleaner backend structure

Course Outcomes Demonstrated

  • Outcome 1: Reliable data workflows
  • Outcome 2: Clear technical communication
  • Outcome 3: Efficient data handling design
  • Outcome 4: Use of MongoDB and Mongoose
  • Outcome 5: Security through validation and error handling

Reflection

This enhancement improved my understanding of database design and its impact on performance and reliability.
