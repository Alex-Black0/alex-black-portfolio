Enhancement Three: Databases

Overview

This enhancement focuses on improving the database layer of the Travlr full-stack application. The goal was to strengthen data validation, improve data integrity, and optimize how data is stored and accessed.

Artifact Description

The artifact used for this enhancement is the Travlr full-stack application developed in CS 465. This application uses MongoDB, Express, Node.js, and Angular to manage trip data through RESTful API endpoints.

Enhancements Implemented

  • Enhanced tripServices.js with data normalization (trimming strings, converting price to number, uppercasing trip codes)
  • Added validation logic for required fields, valid dates, and non-negative pricing
  • Implemented duplicate trip code prevention during create and update operations
  • Updated travlr.js schema:
    • Converted perPerson from String → Number
    • Added validation rules (required, minlength, min)
    • Maintained index on trip code for performance
  • Refactored trips.js controller:
    • Used validated payloads instead of raw request data
    • Improved error handling (400, 404, 500 responses)

Why These Enhancements Matter

  • Data Integrity: Prevents invalid or incomplete data from being stored
  • Performance: Indexed queries improve lookup speed
  • Security: Validation reduces risk of malformed or harmful input
  • Maintainability: Service layer improves code organization

Course Outcomes Demonstrated

  • Designed solutions to store, manipulate, and retrieve data
  • Applied database tools (MongoDB, Mongoose)
  • Implemented a security mindset through validation
  • Improved backend architecture and scalability

Reflection

This enhancement improved my understanding of how database design impacts performance and reliability. I learned the importance of validating and normalizing data before it reaches the database and ensuring that backend changes do not break existing functionality.
