Enhancement One: Software Engineering and Design

Overview

This enhancement focused on improving the software design and engineering quality of the Travlr full-stack application. The primary goal was to make the backend more modular, maintainable, and easier to extend by separating responsibilities between controllers and a dedicated service layer.

Artifact Description

The artifact used for this enhancement is the Travlr full-stack application developed in CS 465. The project uses MongoDB, Express, Angular, and Node.js to manage travel data in a full-stack web application. For this enhancement, I improved the backend architecture by restructuring code responsibilities and introducing a more maintainable service-based design.

Enhancements Implemented

  • Created a dedicated tripService.js file to separate business logic from route controller logic
  • Refactored backend controllers so they call service-layer functions instead of handling all logic directly
  • Improved modularity by assigning clearer responsibilities to controllers, services, and data access logic
  • Reduced code duplication by centralizing reusable trip-related logic in one location
  • Made the backend easier to test, debug, update, and extend in future development

Impact of Enhancements

  • Modularity: The codebase is better organized because each layer has a more focused responsibility
  • Maintainability: Changes to business logic can be made in one place instead of across multiple files
  • Scalability: The application structure now supports future enhancements more effectively
  • Readability: The backend is easier to understand because the flow of responsibilities is clearer

Course Outcomes Demonstrated

  • Outcome 1: Supported collaborative development by improving code organization and making project structure easier for others to follow
  • Outcome 2: Communicated technical design improvements through clearer architecture, documentation, and code separation
  • Outcome 3: Evaluated design trade-offs and improved the application using software engineering principles focused on maintainability and structure
  • Outcome 4: Applied well-founded development practices to create a more robust and extensible backend solution
  • Outcome 5: Strengthened backend structure in a way that supports safer and more controlled handling of application logic

Reflection

This enhancement improved my understanding of how software architecture affects long-term maintainability and project quality. I learned that separating business logic from controller logic creates a cleaner structure and makes the codebase easier to manage. This process reinforced the value of modular design and showed how engineering decisions can improve both developer efficiency and the overall quality of an application.
