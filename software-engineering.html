Enhancement One: Software Engineering and Design

Overview

This enhancement focuses on improving the software design and engineering quality of the Travlr full-stack application. The goal was to make the application more modular, maintainable, and easier to extend by separating responsibilities across the service and controller layers.

Artifact Description

The artifact used for this enhancement is the Travlr full-stack application developed in CS 465. This project uses MongoDB, Express, Angular, and Node.js to manage travel data through a full-stack web application. For this enhancement, I improved the backend structure by introducing a service layer and refactoring controller logic.

Enhancements Implemented

  • Created a dedicated tripService.js file to separate business logic from route controllers
  • Refactored backend controllers to call service-layer functions instead of handling all logic directly
  • Improved modularity by assigning clear responsibilities to each part of the backend architecture
  • Reduced duplication and improved maintainability by centralizing reusable trip-related logic
  • Made the codebase easier to test, update, and extend in future development

Impact of Enhancements

  • Modularity: The application is better organized because controllers and services now have separate responsibilities
  • Maintainability: Changes to business logic can be made in one place instead of across multiple files
  • Scalability: The service-layer structure supports future growth and additional features more easily
  • Readability: The code is easier to understand because each file now has a more focused purpose

Course Outcomes Demonstrated

  • Outcome 1: Supported collaborative development by creating a clearer and more organized project structure
  • Outcome 2: Communicated technical design improvements through code organization and supporting documentation
  • Outcome 3: Evaluated trade-offs in application structure and improved the design using software engineering principles
  • Outcome 4: Applied well-founded development practices to improve code quality and long-term usability
  • Outcome 5: Strengthened the application structure in a way that supports safer and more controlled handling of backend operations

Reflection

This enhancement improved my understanding of how software architecture affects maintainability and long-term project quality. I learned that separating responsibilities into controller and service layers creates a cleaner structure and makes the codebase easier to manage. This process reinforced the value of modular design and showed how engineering decisions can improve both developer efficiency and application quality.
