Enhancement One: Software Engineering and Design

Overview

This enhancement focuses on improving the software design and engineering quality of the Travlr full-stack application. The goal was to make the application more modular, maintainable, and easier to extend by separating responsibilities across the service and controller layers.

Artifact Description

The artifact used for this enhancement is the Travlr full-stack application developed in CS 465. This project uses MongoDB, Express, Angular, and Node.js to manage travel data through a full-stack web application.

Enhancements Implemented

  • Created a dedicated tripService.js file to separate business logic from route controllers
  • Refactored backend controllers to call service-layer functions instead of handling all logic directly
  • Improved modularity and reduced code duplication
  • Improved maintainability and scalability of backend logic

Impact of Enhancements

  • Modularity: Clear separation of responsibilities
  • Maintainability: Easier to update and debug
  • Scalability: Supports future expansion

Course Outcomes Demonstrated

  • Outcome 1: Improved collaboration through structured code organization
  • Outcome 2: Clear communication through modular design
  • Outcome 3: Applied software design principles
  • Outcome 4: Used modern development practices
  • Outcome 5: Improved control of backend logic for safer execution

Reflection

This enhancement strengthened my understanding of software architecture and modular design. I learned how separating concerns improves maintainability and supports long-term development.
